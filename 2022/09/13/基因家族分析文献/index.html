基因家族分析文献

发表于2022-09-13|更新于2022-09-14|Literature
|字数总计:4.1k|阅读时长:12分钟|阅读量:

简介

geneHummus：一个 R 包，用于定义基因家族及其在豆类及其他植物中的表达

在过去的十年中，随着下一代测序技术的快速发展，植物生物技术实验室以可承受的价格引发了一场巨大的革命。很快，这些测序技术和全基因组组装将超越植物计算生物学家，并在植物生物学学科中变得司空见惯。目前可用于非传统植物模型系统（所谓的“孤儿作物”(orphan crops)）的大规模基因组资源使得构建具有植物育种潜在应用的高密度综合物理和遗传连锁图成为可能。新获得的完全测序的植物基因组为比较分析提供了难得的机会，可能揭示基因组生物学和进化的新方面。分析跨物种基因家族的扩张和进化是推断生物学功能的常用方法。迄今为止，植物中基因家族的范围和作用只得到了部分解决，许多基因家族仍有待研究。基因家族的手动识别非常耗时且费力，需要手动和计算分析的迭代过程来识别给定家族的成员，通常结合大量的 BLAST 搜索和手动清理数据。由于基因组序列的日益丰富和对植物基因家族的农艺学兴趣，该领域需要一个清晰、自动化的注释工具。

在这里，我们介绍了geneHummus 包，这是一个基于R 的管道，用于识别和表征植物基因家族。该管道的影响来自于动手注释时间的减少以及从 RefSeq 数据库中仅提取蛋白质并提供基于 SPARCLE 的保守域架构的高特异性和敏感性。作为案例研究，我们专注于 Cicer arietinum（鹰嘴豆）和其他豆类中的生长素受体因子基因 (ARF) 家族。

背景

通过使用下一代测序 (NGS) 技术，研究人员已经大量增加了存储在公共数据库中的核苷酸序列数量 [1]。这已经彻底改变了包括植物科学在内的众多领域（图（图 1）.1）。然而，植物科学领域的一个瓶颈是蛋白质序列的注释及其功能的表征。识别重要蛋白质的功能可用于改善农艺性能，例如改变植物对特定环境压力因素（如干旱或高温）的抗性或耐受性。推断未知蛋白质功能的一种方法是在具有已知功能的蛋白质中鉴定保守序列，这在同源性可以暗示保守的生化功能的程度上是有用的。

植物激素生长素（吲哚-3-乙酸）几乎是植物生长发育各个方面的关键调节剂。作为生长素信号通路的核心作用，生长素反应因子 (ARF) 多基因家族存在于陆地植物的所有主要分支中 。考虑到 ARF 家族成员作为植物生长和发育过程调节剂的重要作用，在过去几年中，人们对研究一年生草本植物 和多年生木本植物 中的 ARF 家族产生了相当大的兴趣。 ARF 的表征通常可以深入了解基因组结构 、基因组中的位点分布 、序列同源性 、系统发育历史 以及发育和/或生物/非生物过程中压力的基因表达模式。 ARF 家族成员的有效表达可以极大地提高可用于提高农艺性能的有用基因组靶标的数量，以及它们表征的速度。在豆科植物中这样做可能会对营养食品的供应和生态系统恢复能力产生巨大影响 。

目前，植物基因家族的鉴定和表征既费力又费时。它需要计算分析的迭代过程来识别给定家族的基因家族成员。该过程主要基于基本局部对齐搜索工具 (BLAST) 搜索，其中散布着手动管理和修剪。由于序列数量的增加和对植物基因家族的农艺兴趣，这个过程可以从自动化工具中受益。尽管已经开发了许多生物信息学工具来识别同源蛋白质，但这些工具在很大程度上仅限于完全测序的动物基因组的静态数据库，因此需要一种可以用于植物基因组的工具。

为了满足这一需求，我们开发了geneHummus，一种新型的R 包，可有效识别植物基因家族的成员（种质）。通过查询 SPARCLE 和 RefSeq 数据库，geneHummus 可以快速地将架构标识符从分类界的整个或草图基因组中分离出来，并随着新序列的加入而更新。此外，geneHummus 简化了下游分析，例如系统发育构建和基因表达谱。作为一个案例研究，我们专注于 Cicer arietinum（鹰嘴豆）和其他豆科植物中的生长素受体因子基因 (ARF) 家族，之前已经手动注释了鹰嘴豆基因组中的 ARF 家族 ，因此有一个黄金标准数据集可用于比较我们的管道结果。在我们关于 ARF 基因家族的案例研究中，geneHummus 的功能允许与系统发育和表达谱的集成工作流程。我们预计我们的管道应该适用于研究任何植物基因家族，以及可能的其他基因分类家族，从而大大提高基因组数据处理的速度和易用性。

实施

geneHummus 是作为 R 包实现的，需要一组最小的依赖项（dplyr [16]、stringr [17]、rentrez [18]、httr、utils 和 curl 包），这些依赖项会根据需要从 CRAN 存储库 [19] 自动下载.在本节中，我们将详细描述geneHummus 的实现。我们的 Github 存储库 [20] 还提供了描述 ARF 案例研究的其他文档。该管道专为具有最少编程经验的生物学家最终用户设计，使用开源和免费软件（R、NCBI 工具）指导用户完成基因家族的识别、表征和表达分析。植物基因家族的特征在于共同的蛋白质结构。定义给定的家族结构从文献中已知。例如，ARF 基因家族的隐马尔可夫模型 (HMM) 谱是 B3 DNA 结合域 (B3)、AUX_RESP 和 AUX/IAA，它们对应于保守域 Pfam 02362、Pfam 06507 和 Pfam 02309。管道首先将保守域登录号定义为对 NCBI 的 SPARCLE 数据库的查询（图（图 2）.2）。然后，您可以获得每个保守域的 SPARCLE 体系结构标识符 (id)，并仅提取或过滤那些表征 ARF 基因家族的体系结构。接下来，检索每个候选 ARF 架构的蛋白质电子 ID。根据您的数据集，此步骤可能需要几秒钟到 3–5 分钟。请注意，如果您有很长的蛋白质 ID 列表，当您尝试与 NCBI E-utilities 交互时，您可能会收到 414 错误。 geneHummus 对元素进行子集化（每个列表最多 300 个 id），因此函数可以正常工作。检索到的蛋白质 ID 由感兴趣的分类 ID [21] 过滤，在本案例研究中是豆科植物 ID。 genehummus 仅返回由 RefSeq 数据库托管的蛋白质 ID。最后，将电子标识转换为蛋白质信息。在这一点上，我们可能已经从豆科植物中鉴定了整套 ARF 蛋白种质。从种质中下载氨基酸序列很简单。下载这些序列后，相关信息可用于系统发育和表达研究。

结果

已经开发了许多方法来预测不同蛋白质的功能。国家生物技术信息中心 (NCBI) 最近开发的资源亚科蛋白质结构标记引擎 (SPARCLE) 就是这样一种方法。 SPARCLE 可以根据蛋白质序列的特征域结构对其进行分组，从而帮助对蛋白质序列进行功能表征。我们搜索了 SPARCLE 数据库以获得基于定义 ARF 基因家族的保守结构域的整套分子结构。然后，我们从 SPARCLE 中过滤数据以选择豆科（豆科）的分类群和源数据库（RefSeq）。在这些过滤器之后，我们获得了 560 多种不同的 ARF 豆科植物蛋白，这些蛋白由~ 330 个基因位点编码。在按物种分离这些结果后，我们的管道在鹰嘴豆基因组 (Cicer arietinum) 中鉴定了 24 个 ARF 蛋白，重现了之前通过迭代详尽的 BLAST 搜索获得的结果 。这些结果验证了geneHummus方法，该方法提供了一种在不到6分钟的时间内产生这些结果的自动化方法，而详尽的BLAST方法需要大量的手动管理和超过6个月的工作。此外，geneHummus 管道还返回了在这个 6 min 处理窗口中其他 9 个豆科植物中 ARF 蛋白的数量。有趣的是，ARF 蛋白的数量在同一属的不同物种中是相似的（Arachis duranesis 和 Arachis ipaensis；Vigna angularis 和 Vigna radiata），正如预期的那样。此外，已知由于全基因组复制事件而拥有大量扩展的旁系同源基因的物种，例如甘氨酸和羽扇豆谱系 ，在转录本数量和 ARF 基因座数量方面均显示出最高值。我们发现geneHummus 具有特异性和敏感性，通过详尽的搜索鉴定了先前在其他豆科植物研究中报道的表征ARF 家族的相同序列，例如M. truncatula 或大豆。但我们也发现了比以往报道的更多的 ARF 种质，因为geneHummus 使用了 GenBank 提供的最新更新的 RefSeq 数据库版。所有这些结果都增加了对我们方法有效性的信心。为了使用我们的管道查找不同豆科植物中的 ARF，我们开发了一个交互式shiny应用程序来访问这些数据 。在运行geneHummus管道后，有兴趣为其基因家族和感兴趣的分类群加载相关表的研究人员可以克隆和修改以轻松分享他们的结果。

geneHummus 是一个灵活的工具，可以处理几乎任何基因家族和几乎任何植物物种。 HomoloGene 是一个自动化系统，用于检测 21 个完全测序的真核基因组中的同源物；但是，它不够灵活，无法应用于基因组草图。 MultiMSOAR 和 OrthoMCL 是开发用于在不同基因组中寻找直系同源群的其他工具。这些工具仅限于包含的软件版本和基因组，不包括鹰嘴豆基因组或其他豆科植物。 Ensembl 植物还提供了用于提取目标基因家族的编程工具，但同样受限于有限数量的基因组和浏览器托管的序列。虽然这些工具非常适合其预期目的，但它们无法满足对鉴定保守蛋白质结构感兴趣的植物遗传学家的需求。

geneHummus 在几个方面超越了现有工具。它通过从 NCBI 数据中搜索蛋白质结构来识别用于进一步分析的序列，然后在不需要下载基因或蛋白质序列的情况下检索基因家族。它使用最新更新的参考序列，因此在涵盖整个和部分基因组的物种和科的广度方面更加全面，并且比静态数据库更准确。尽管geneHummus 要求最终用户对R 编程语言有最低限度的熟悉，但通过四个函数的顺序调用，管道在用户友好和快速的环境中识别给定的基因家族。如果序列由 GenBank 托管，geneHummus 的主要优势在于用户可以将其即时应用于任何基因组，并且可以对其进行定制以适用于豆科植物以外的其他重要农艺分类家族。例如，在安装软件包时，用户可以访问多个对象，这些对象包含十字花科、葫芦科、蔷薇科和茄科的分类 ID。这也可以为其他家庭定制，这实际上使基因鹰嘴豆泥成为植物研究界的有用工具。

可以对geneHummus 结果进行大量下游分析和应用。一个例子是系统发育分析。基于 ARF 蛋白的保守结构域，我们探索并描绘了整个数据集之间的序列关系。来自裸子植物谱系的两种 ARF 蛋白被列为外群物种。裸子植物已被解决为被子植物的姐妹群。它们与大约 3.1 亿年前的最近共同祖先背道而驰 。蛋白质序列的系统发育分布表明，所有 ARF 序列都属于两个主要组（I 和 II），具有良好的引导值。 I 组数量最多，可以进一步细分为包含拟南芥序列的直系同源物的簇，这些序列定义了众所周知的进化枝 AtARF3/4-like、ARF12-like、ARF10/16-like 和 ARF17-like [32]。第二组包含类似 AtARF5 的集群。第二组中的第二个簇不包含任何拟南芥直系同源物（数据未显示），这意味着该进化枝是通过豆科植物物种的保守功能的长期进化获得的。我们将那些基于高引导值 (> 65%) 聚集在一起的蛋白质标记为姐妹对。与涉及鹰嘴豆的姐妹对相关，系统发育结构有 8 对姐妹对（七对 C. arietinum-M. truncatula 和一对 C. arietinum-L. angustifolius）。我们没有观察到两个鹰嘴豆 ARF 蛋白之间的任何姐妹对。这是一个有趣的进化模式。鹰嘴豆从 M. truncatula 分化而来~ 10-2000 万年前 。缺乏鹰嘴豆姐妹对表明最近的重复（在鹰嘴豆和紫花苜蓿分离后）在 ARF 鹰嘴豆家族的扩张中发挥了非常有限的作用（如果有的话），或者重复的蛋白质没有太大变化，因为这两个物种有一个共同的祖先.这两个假设也是合理的。

应用基因鹰嘴豆泥管道，我们对豆科植物中整个 ARF 蛋白质数据集的结构和系统发育进行了表征。作为一个案例研究，我们还描述了 Cicer arietinum 中基因家族的表达谱。该管道的实用性源于手动注释时间的减少以及从 RefSeq 数据库中提取蛋白质的高特异性和敏感性，并提供与其他 NCBI 工具套件的交互，例如基于最近开发的 NCBI 的保守域架构资源 SPARCLE 和对齐器 Magic-BLAST。 geneHummus 是一种强大的工具，可用于鉴定可用于系统发育分析的基因家族序列。结果表明，大多数蛋白质在跨属中都非常保守，具有丰富的多物种进化枝。这表明这些蛋白质参与了常见的基本细胞活动。该直系学信息可用于根据蛋白质在另一个属中的已知功能来推断给定物种中先前未表征的蛋白质的功能。这是比较基因组学的一种特别有效的方法。一旦确定了序列，鉴于可用 SRA 文库适用于多种组织和条件的能力，用户可以通过使用基于 Magic-BLAST 的差异表达分析来识别特定条件下感兴趣的基因，从而充分利用管道.该工具将帮助研究人员发现基因，并特别适用于植物育种计划以及其他应用。我们的流程适用于所有测试的植物基因组的 ARF 家族检测，并且应该适用于其他基因家族，极大地提高了基因组数据处理的速度和易用性。
