简介

总是感觉自己学到的R语言不太扎实，就找了一本书，想系统深入的学习R语言，了解该语言的工作原理并学习解决各种问题的新策略。为此记录自己的学习心得。本学习心得主要基于R的大佬Hadley Wickham的《Advanced R》

Hadley认为作为一名R程序员最重要的两点：

  • 阅读源代码（scientific mindset）：查看您最常使用的函数和包的源代码
  • 采用科学的思维方式（scientific mindset）：如果你不明白某件事是如何工作的，你应该提出一个假设，设计一些实验，运行它们，并记录结果。

修改时复制

向量

向量有两种形式：原子向量和列表20。它们的元素类型不同：对于原子向量，所有元素必须具有相同的类型；对于列表，元素可以有不同的类型。虽然NULL不是向量，但 NULL 与向量密切相关，并且通常充当通用零长度向量的角色。

每个向量还可以具有属性，您可以将其视为任意元数据的命名列表。有两个属性特别重要。维度属性将向量转换为矩阵和数组，类属性为 S3 对象系统提供支持。

原子向量

原子向量有四种主要类型：逻辑(logical)、整数(integer)、双精度(double)和字符(character)（包含字符串）。整数和双精度向量统称为数字(numeric)向量。有两种罕见的类型：复杂(complex)和原始(raw)。统计中很少需要复杂向量，而原始向量是一种特殊类型，仅在处理二进制数据时才需要。

标量

四种主要类型中的每一种都有一个特殊的语法来创建一个单独的值，也就是一个标量

  • 逻辑可以全写（TRUE 或 FALSE），或缩写（T 或 F）。
  • 双精度可以指定为十进制 (0.1234)、科学计数 (1.23e4) 或十六进制 (0xcafe) 形式。双精度数具有三个独特的特殊值：Inf、-Inf 和 NaN（不是数字）。这些是浮点标准定义的特殊值。
  • 整数的写法类似于双精度数，但后面必须跟 L23（1234L、1e4L 或 0xcafeL），并且不能包含小数值。 R的整型等同于C语言的long integer,C语言代码中会使用后缀L或者l来迫使数值伟long integer，但是R语言i被使用到complex number,所以就用L
  • 字符串由 " (“hi”) 或 ’ (‘bye’) 包围。特殊字符用 \ 转义；有关完整详细信息，请参阅 ?Quotes。

缺失值

R 表示缺失值或未知值，具有特殊标记值：NA（不适用的缩写）。大多数涉及缺失值的计算将返回另一个缺失值。这条规则只有少数例外(当某些身份对所有可能的输入都成立时，就会发生这些情况：)。

注意：从技术上讲，有四个缺失值，每个原子类型一个：NA（逻辑）、NA_integer_（整数）、NA_real_（双精度）和 NA_character_（字符）。这种区别通常并不重要，因为 NA 会在需要时自动强制转换为正确的类型。

测试和强制

通常，您可以使用 is.*() 函数测试向量是否属于给定类型，但需要小心使用这些函数。 is.logical()、is.integer()、is.double() 和 is.character() 执行您可能期望的操作：它们测试向量是字符、双精度、整数还是逻辑向量。避免使用 is.vector()、is.atomic() 和 is.numeric()：它们不会测试您是否有向量、原子向量或数值向量；您需要仔细阅读文档以了解它们的实际作用。

对于原子向量，类型是整个向量的一个属性：所有元素必须是相同的类型。当您尝试组合不同的类型时，它们将被强制以固定顺序：字符 → 双精度 → 整数 → 逻辑。例如，组合一个字符和一个整数会产生一个字符：

S3原子向量

最重要的向量属性之一是类，它是 S3 对象系统的基础。拥有类属性会将对象转换为 S3 对象，这意味着当传递给通用函数时，它的行为将与常规向量不同。每个 S3 对象都建立在基本类型之上，并且通常将附加信息存储在其他属性中，这个将会在R学习笔记之面向对象中介绍。

我们将讨论基础 R 中使用的四个重要的 S3 向量：

  • 分类数据，其中值来自记录在因子向量中的一组固定水平
  • 日期（具有日期分辨率），记录在日期向量中
  • 存储在 POSIXct 向量中的日期时间（具有秒或亚秒分辨率）
  • 持续时间，存储在 difftime 向量中。
因子

因子是只能包含预定义值的向量。它用于存储分类数据。因子建立在具有两个属性的整数向量之上：

类，“因子”，它使其行为不同于常规整数向量，以及定义一组允许值的级别。

当您知道一组可能的值但它们并不全部存在于给定的数据集中时，因子很有用。与字符向量相比，当您将一个因子制成表格时，您将获得所有类别的计数，甚至是未观察到的类别：

有序因子（Ordered factors）是因子的微小变体。通常，它们的行为类似于常规因子，但级别的顺序是有意义的（低、中、高）（一些建模和可视化功能自动利用的属性）。

许多基本 R 函数（如 read.csv() 和 data.frame()）会自动将字符向量转换为因子。这是次优的，因为这些函数无法知道所有可能级别的集合或其正确顺序：级别是理论或实验设计的属性，而不是数据的属性。相反，使用参数 stringsAsFactors = FALSE 来抑制这种行为，然后使用您对“理论”数据的了解将字符向量手动转换为因子。

虽然因子看起来像（并且通常表现得像）字符向量，但它们是建立在整数之上的。因此，在将它们视为字符串时要小心。一些字符串方法（如 gsub() 和 grepl()）会自动将因子强制转换为字符串，其他（如 nchar()）会抛出错误，还有一些（如 c()）会使用底层整数值。因此，如果您需要类似字符串的行为，通常最好将因子显式转换为字符向量。

日期

日期向量建立在双向量之上。它们有“日期”类，没有其他属性：

double 的值（通过剥离类可以看出），表示自 1970-01-01 以来的天数

日期时间

Base R 提供两种存储日期时间信息的方法，POSIXct 和 POSIXlt。这些是公认的奇怪名称：“POSIX”是 Portable Operating System Interface 的缩写，它是一系列跨平台标准。 “ct”代表日历时间（C 中的 time_t 类型），“lt”代表本地时间（C 中的 struct tm 类型）。

POSIXct，因为它最简单，构建在原子向量之上，并且最适合在数据帧中使用。 POSIXct 向量建立在双向量之上，其中的值表示自 1970 年 1 月 1 日以来的秒数。

tzone 属性仅控制日期时间的格式；它不控制向量表示的时间瞬间。请注意，如果是午夜，则不会打印时间。

列表

  • 列表比原子向量更复杂：每个元素可以是任何类型，而不仅仅是向量。从技术上讲，列表的每个元素实际上是相同的类型，因为每个元素实际上是对另一个对象的引用，该对象可以是任何类型。
  • 因为列表的元素是引用，所以创建列表不涉及将组件复制到列表中。因此，列表的总大小可能比您预期的要小。
  • 列表有时称为递归向量，因为列表可以包含其他列表。这使得它们从根本上不同于原子向量。

数据框和tibbles

如果您在 R 中进行数据分析，您将使用数据框。数据框是具有（列）names，row.names，及其类(data.frame)属性的向量的命名列表：

与常规列表相比，数据框有一个额外的约束：每个向量的长度必须相同。这为数据框提供了矩形结构，并解释了为什么它们共享矩阵和列表的属性：

  • 一个数据框有 rownames() 和 colnames()。数据框的 names() 是列名。
  • 一个数据框有 nrow() 行和 ncol() 列。
  • 数据框的 length() 给出了列数。

数据框是 R 中最大和最重要的思想之一，也是使 R 与其他编程语言不同的东西之一。然而，自创建以来的 20 多年中，人们使用 R 的方式发生了变化，一些在创建数据框时有意义的设计决策现在令人沮丧。 这种挫败感导致了 tibble 的创建，对数据框的现代重新构想。 Tibbles 被设计为（尽可能地）替代数据框来解决这些问题。总结主要区别的一种简洁而有趣的方式是，tibble 是懒惰和粗暴的：他们做的更少，抱怨的更多。Tibbles 由 tibble 包提供，并与数据帧共享相同的结构。唯一的区别是类向量更长，并且包含 tbl_df。这允许 tibbles 以我们将在下面讨论的关键方式表现不同。

虽然数据框（或 tibble）的每个元素都必须具有相同的长度，但 data.frame() 和 tibble() 都将回收较短的输入。然而，虽然数据框会自动回收最长列的整数倍的列，但 tibbles 只会回收长度为 1 的向量。

行名

数据框允许您使用名称标记每一行，一个仅包含唯一值的字符向量：

如果您将数据框视为像矩阵这样的二维结构，行名称自然会出现：列（变量）具有名称，因此行（观察值）也应如此。大多数矩阵都是数字的，所以有一个地方来存储字符标签很重要。但是这种与矩阵的类比具有误导性，因为矩阵具有数据框所没有的重要属性：矩阵是可转置的。在矩阵中，行和列是可以互换的，转置一个矩阵会给你另一个矩阵（再次转置给你原始矩阵）。然而，对于数据框，行和列是不可互换的：数据框的转置不是数据框。

行名不受欢迎的原因有以下三个：

  • 元数据是数据，因此以与其他数据不同的方式存储它从根本上说是个坏主意。这也意味着您需要学习一套新的工具来处理行名；你不能使用你已经知道的关于操作列的知识。
  • 行名称对于标记行来说是一种糟糕的抽象，因为它们仅在一行可以由单个字符串标识时才起作用。这在很多情况下都会失败，例如，当您想通过非字符向量（例如时间点）或多个向量（例如位置，按纬度和经度编码）来识别行时。
  • 行名必须是唯一的，因此行的任何重复（例如来自bootstrapping）都会创建新的行名。如果要匹配转换前后的行，则需要执行复杂的字符串命令。

由于这些原因，tibble不支持行名称。相反，tibble 包提供了一些工具，可以使用 rownames_to_column() 或 as_tibble() 中的 rownames 参数轻松地将行名转换为常规列：

输出

tibble和数据框之间最明显的区别之一是它们的输出方式。使用 dplyr 包中包含的示例数据集强调一些最大的差异：

  • Tibbles 仅显示前 10 行和所有适合屏幕的列。附加列显示在底部。
  • 每列都标有其类型，缩写为三个或四个字母。
  • 宽列被截断以避免单个长字符串占据整行。 （这仍然是一项正在进行的工作：在显示尽可能多的列和完整显示列之间进行权衡。）
  • 在支持它的控制台环境中使用颜色时，会明智地使用颜色来突出重要信息，并淡化补充细节。
取子集

可以将数据框或 tibble 子集化为一维结构（其行为类似于列表）或二维结构（其行为类似于矩阵）。

数据框有两种不受欢迎的子集行为：

  • 当您使用 df[, vars] 对列进行子集化时，如果 vars 选择一个变量，您将获得一个向量，否则您将获得一个数据框。这是在函数中使用 [ 时常见的错误来源，除非您始终记得使用 df[, vars, drop = FALSE]。

  • 当您尝试使用 dfx提取单个列并且没有列x时，数据框将改为选择以x开头的任何变量。如果没有变量以x开头，则dfx 提取单个列并且没有列 x 时，数据框将改为选择以 x 开头的任何变量。如果没有变量以 x 开头，则 dfx提取单个列并且没有列x时，数据框将改为选择以x开头的任何变量。如果没有变量以x开头，则dfx 将返回 NULL。这使得选择错误变量或选择不存在的变量变得容易。

Tibbles 调整了这些行为，因此 [ 总是返回一个 tibble，而 $ 不进行部分匹配并在找不到变量时发出警告（这就是使 tibbles 生硬的原因）

tibble 坚持从 [ 返回数据帧可能会导致遗留代码（legacy code）出现问题，这些代码通常使用 df[, “col”] 来提取单个列。如果你想要一个单列，建议使用 df[[“col”]]。这清楚地传达了您的意图，并且适用于数据框和tibble。
